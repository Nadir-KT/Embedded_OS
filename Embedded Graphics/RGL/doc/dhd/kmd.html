Kernel Mode API

 

There is only a single kernel mode driver (KMD) instance. The KMD should be initialized/closed at OS boot/shutdown time. It can serve any number of hardware instances (to the compiletime limit DHD_MAX_GPU_COUNT) and any number of applications. Each application has it's own user mode driver (UMD) called a 'client' from the kernel mode's point of view. 

 

While the binding between application and front end as well as between front end and UMD is very tight (direct linking), the binding between kernel and user mode can be rather loose. 

A transition from UMD to KMD will usually (this is highly OS dependent) require messages to be sent by the OS. So none of the kernel mode API functions should be called frequently.
