D/AVE HD Driver - Internal Documentation
dhd_draw
Rendering Functions | See Also | Send Feedback
Summary

Render a single primitive.

Syntax
void dhd_draw(dhd_handle_t a_device, dhd_uint32_t a_edgecontrol, dhd_uint32_t a_edgecount, dhd_drawattribute_flag_t a_attributes);
Description

The data for edges and attribute (e.g UV) mappings is read from the respective buffers (see dhd_set_edge_buffer, dhd_set_uv_buffer etc.). The amount of data to be read from attribute buffers is specified using the a_attributes flag, which is usually a combination of bits from dhd_drawattribute_flag_t. An alternative to the enums from dhd_drawattribute_flag_t are the DHD_DRAW_UV(n), DHD_DRAW_RHW(n) and DHD_DRAW_COL(n) macros which can be used if the number of UV pairs per primitive is a dynamic value. 

 

Note that the attribute parameter specifies how much data is read from the attribute buffers per primitive, it does not trigger usage of that attribute (e.g. texturing). Also if an attribute has been set explicitly (by e.g. dhd_set_uv) it should not be listed as an attribute flag, as that would overwrite the specified value with data from a buffer.

Parameters
Parameters 
Description 
a_device 
Driver context handle (as returned by dhd_init) 
a_edgecontrol 
Bits that describe the active geometry edges. (see Edge Macros) 
a_edgecount 
Number of edges to read from edge buffer (as specified by dhd_set_edge_buffer) 
a_attributes 
Number of attributes to be read from attribute buffers (see dhd_drawattribute_flag_t)
