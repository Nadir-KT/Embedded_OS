dhd_alloc_buffer
Buffer Functions | See Also | Send Feedback
Summary

Allocate video memory for a buffer.

Syntax
dhd_enum_t dhd_alloc_buffer(dhd_handle_t a_device, dhd_buffer_t * a_buffer, dhd_buffer_flags_t a_flags);
Description

The a_buffer struct must be initialized prior to calling this function. If successful the buffer can be used for other operations (e.g. data uploads / downloads). 

The following fields of dhd_buffer_t have to be initialized prior to calling alloc_buffer:

  • m_width, m_height
  • m_pitch
  • m_bpp
  • m_usage

 

For subbyte formats, m_format has also to be initialized. 

For the most common cases of a simple rectangular buffer that is to be used as texture, framebuffer or zsabuffer the function dhd_alloc_buffer_simple can be used instead, which does not need an initialzed buffer description struct.

Parameters
Parameters 
Description 
a_device 
Driver context handle (as returned by dhd_init) 
a_buffer 
Pointer to an initialized dhd_buffer_t struct 
a_flags 
A combination of dhd_buffer_flags_t indicating how the buffer is going to be accessed 
Returns

E_DHD_OK if successful E_DHD_ERROR_INVALID_ARGUMENT in case of an invalid input parameter E_DHD_ERROR_NO_MEMORY if there was no large enough continuous piece of video memory E_DHD_ERROR_INTERNAL if internal structures were inconsistent
