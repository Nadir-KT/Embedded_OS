dhd_isbusy_marked
General Functions | See Also | Send Feedback
Summary

Asynchronously check a marked position.

Syntax
dhd_bool_t dhd_isbusy_marked(dhd_handle_t a_device, const dhd_mark_t * a_mark);
Description

Similar to dhd_finish_marked this function refers to a specific position inside the command stream (see dhd_sync_mark). It returns true when this position is still busy / has not yet been successfully executed by the HW. 

While calling dhd_isbusy_marked in a loop until it returns false is possible, it is recommended to use dhd_finish_marked instead in this case as it will let the calling thread sleep while waiting.

Parameters
Parameters 
Description 
a_device 
Driver context handle (as returned by dhd_init) 
a_mark 
Pointer to a sync mark (has to be filled out by dhd_sync_mark) 
Returns

DHD_TRUE when the job is still busy (being executed or waiting for execution) DHD_FALSE when the job is not busy or an error occured (error only if a_device is invalid, a_mark==0 or a_mark->m_jobid==0)

See Also

dhd_finish_marked, dhd_sync_mark
