GlobalSettings
Topic 
Summary 
DHD_CACHELINE_SIZE 
Size of one CPU cacheline in bytes. 
DHD_DEBUG_GUARDED_LOOPS 
Enables debug mode for guarded loops (guard values can be set on runtime). 
DHD_DEBUG_MODE 
Add additional parameter validation (does not work with inline interface). 
DHD_DEBUG_STATE_CHECKSUM 
Add checksumming to dhd_get_state and dhd_set_state functions to detect corrupted states. 
DHD_DEVICE_DATA_SIZE 
Size of the internal device data. 
DHD_GUARDED_LOOPS 
Enable loop guards (prevents infinite loops). 
DHD_INLINE_INTERFACE 
Set if usermode API function should be inlined where possible. 
DHD_JOB_CHECKSUM_SUPPORTED 
Enable support for stream checksumming 
DHD_KERNEL_DEBUG 
Add kernel internal debugging output. 
DHD_KERNEL_DEBUG_ERROR 
Add debug output in case of an hardware error. 
DHD_KERNEL_DEBUG_MEMORY 
Add debug output for video memory heap debugging (slow). 
DHD_KERNEL_DEBUG_MEMORY_VERBOSE 
Increase verbosity level of DHD_KERNEL_DEBUG_MEMORY output. 
DHD_KERNEL_DEBUG_SCHEDULE 
Add debug output for each job schedule related action (slow). 
DHD_KERNEL_DEBUG_TRACEVCD 
Add support to enable event traces (requires tracevcd module). 
DHD_KERNEL_DUMB_ALLOCATOR 
Use a simple O(n) video memory allocator instead of the optimized O(1) code (slow). 
DHD_KERNEL_MEMORY_PATTERN 
Fill video memory with special debug patterns (does work only if CPU can read/write videomemory). 
DHD_KERNEL_NON_LEVEL_IRQ 
Support HW with not level sensitive interrupt lines (requires extra handling of IRQs). 
DHD_KERNEL_SOFT_IRQ 
IRQ can be interrupted by normal code (only on softdave / emulated IRQs) 
DHD_KERNEL_SYNCHRONOUS_ONLY 
When defined all kernel calls are blocking. 
DHD_KERNEL_TLIST_SUPPORT 
Add kernel tlist debugging support (must still be enabled by user mode). 
DHD_KERNEL_USE_POOLING 
Put all kernel memory allocations through the kernel pool allocator (no dynamic allocations). 
DHD_KERNEL_VALLOC_HOOK 
Report video ram allocations / frees to the platform specific layer. 
DHD_MAX_BUFFER_HEIGHT 
Maximum accepted buffer height (used for buffer sanity checks) 
DHD_MAX_BUFFER_WIDTH 
Maximum accepted buffer width (used for buffer sanity checks) 
DHD_MAX_CONCURRENT_WAITS 
Number of clients that can simultaneously block on wait_for_job (maximum size of the wait queue). 
DHD_MAX_CPU_REGISTER_ACCESS 
Number of registers (32bit) accessible by the CPU. 
DHD_MAX_DEBUG_BUFFER_SIZE 
Size of debug message buffer, must be larger than message_size*2 (only used when DHD_KERNEL_DEBUG is active). 
DHD_MAX_DEBUG_MESSAGE_SIZE 
Maximum number of characters (excluding /0) per note. 
DHD_MAX_GPU_COUNT 
Maximum kernel mode driver instance count. 
DHD_MAX_KERNEL_CLIENTS 
Maximum number of concurrent kernel clients / user threads (only used when DHD_KERNEL_USE_POOLING is active). 
DHD_MAX_USER_BUFFERS 
Maximum number of buffers per application (only used when DHD_USER_USE_POOLING is active). 
DHD_MAX_USER_JOBS 
Maximum number of jobs per application (only used when DHD_USER_USE_POOLING is active). 
DHD_MAX_USER_SHADOW 
Maximum amount of shadow buffer space for non mappable buffers (only used when DHD_USER_USE_POOLING is active). 
DHD_MAX_VIDMEM_CHUNKS 
Maximum number of vidmem chunks (each chunk can serve 64 blocks) (only used when DHD_KERNEL_USE_POOLING is active). 
DHD_MEMORY_MMU 
Enable support for memory managed systems (architectures using a MMU) 
DHD_MEMORY_NO_UMA 
Enable support for architectures without a uniformed memory 
DHD_SUPPORT_CASSERT 
Enable compiletime asserts (only disable these if your compiler does not support the involved macros). 
DHD_TLIST_ALIGN 
Additional memory alignment for TLIST debug output 
DHD_USER_USE_POOLING 
Put all user mode allocations through a pool allocator (not recommended). 
DHD_VIDMEM_MINALIGN_LOG2 
Log2 of the minimum possible alignment for vido memory allocations.
