Video Memory

 

All video ram allocations, done by a certain kernel instance, are done from the memory region specified when initializing the instance. The driver will not use any address outside the range of vidmem_base_address .. vidmem_base_address + vidmem_size - 1. It will also not try to access the video ram in any way using the CPU. Mapping video ram addresses (when possible) or transfering data from or to video ram is implemented in the platform specific code. 

 

The driver will make sure that all allocations are aligned correctly (matching the HW bus width), even if the given base address is not aligned. 

 

While for CPU heaps meta information (like freelists and boundary tags) is usually stored directly inside the heap, the GPU heap all of this is stored externally to avoid CPU access to GPU memory. In order to still provide efficient free block coalescing, the GPU heap manages memory chunks not only by their address (like malloc does) but uses a struct (dhd_vidmem_t), that contains all necessary information. Allocation consequently returns a struct instead of a single pointer. 

The application usually does not access the dhd_vidmem_t struct members directly. It will only use it as a token to identify a memory block.
