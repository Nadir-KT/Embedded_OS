dhd_init

Summary

Create a driver context.

Syntax
dhd_handle_t dhd_init(dhd_device_t * a_device, dhd_uint32_t a_hw_instance, dhd_enum_t a_flags);
Description

All other driver functions require a valid driver context to work. All contexts are separated from each other and have their own virtual view on the bound HW. The caller has to provide the memory to store the context in and ensure it remains valid until all work on the context is done (dhd_shutdown has been called).

Parameters
Parameters 
Description 
a_device 
space to construct the driver context in 
a_hw_instance 
integer selecting the HW instance to work on 
a_flags 
currently unsued (set to 0) 
Detailed Information

Note that dhd_init may not allocate videomemory, as these early allocations can not be recorded by dhd_tlist functions.

Returns

A driver context handle or DHD_INVALID_HANDLE in case of an error
