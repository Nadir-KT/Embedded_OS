D/AVE HD Driver - Internal Documentation
dhd_lock_buffer
Buffer Functions | Send Feedback
Summary

Lock the entire buffer for CPU access.

Syntax
dhd_enum_t dhd_lock_buffer(dhd_handle_t a_device, dhd_buffer_t * a_buffer, dhd_enum_t a_access);
Description

After a lock has been aquired the CPU side address can be received using a call to dhd_get_lockedbuffer_ptr.

Parameters
Parameters 
Description 
a_device 
Driver context handle (as returned by dhd_init) 
a_buffer 
Pointer to an allocated dhd_buffer_t (see dhd_alloc_buffer) 
a_access 
The required access level (either E_DHD_READ_ACCESS or E_DHD_WRITE_ACCESS or both) 
Returns

E_DHD_OK if successful E_DHD_ERROR_INVALID_ARGUMENT in case of an invalid input parameter E_DHD_ERROR_NO_BUFFER the buffer is invalid or has not yet been allocated E_DHD_ERROR_ALREADY_LOCKED when the buffer was already locked
