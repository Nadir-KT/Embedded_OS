D/AVE HD Driver - Internal Documentation
dhd_sync_mark
General Functions | See Also | Send Feedback
Summary

Place a mark in the command stream and return a reference to the mark.

Syntax
dhd_enum_t dhd_sync_mark(dhd_handle_t a_device, dhd_mark_t * a_result);
Description

The CPU can check whether the mark has been reached by the GPU or not afterwards. Use dhd_finish_marked to wait until the GPU has reached a mark. Use dhd_isbusy_marked to query if a mark has been reached or not without waiting. Placing the mark does a pipeline wait first and a flush afterwards, so doing an explicit dhd_flush after dhd_sync_mark is not necessary. Note that because marked sync points require a synchronization of all HW pipelines even if the CPU never actually waits for the mark, using a lot of marks can impact GPU performance. An alternative to find out if the GPU has reached a certain point of the stream is to use dhd_sync_signal.

Parameters
Parameters 
Description 
a_device 
Driver context handle (as returned by dhd_init) 
a_result 
Pointer to a dhd_mark_t that will receive the ID required to wait for the placed mark 
See Also

dhd_finish_marked, dhd_isbusy_marked, dhd_sync_signal
