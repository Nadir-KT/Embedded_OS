D/AVE HD Driver - Internal Documentation
dhd_convert_buffer
Conversion Utility Functions | Send Feedback
Summary

Convert a texture to another format or swizzle.

Syntax
dhd_enum_t dhd_convert_buffer(dhd_handle_t a_device, const dhd_buffer_t * a_input, dhd_buffer_t * a_output, dhd_address_mode_t a_mode);
Description

This function will create a new texture with the same content as the given source but a different texel format and/or swizzle mode. The desired destination format and bpp have to be initialized in the a_output buffer struct. All other elements will be filled out by dhd_convert_buffer. The target buffer pitch and physical size might have to be larger than the original to enable the desired swizzle mode. dhd_convert_buffer will determine the smallest size that fullfills all requirements and use that. 

 

Note that dhd_convert_buffer will allocate the output_buffer using dhd_alloc_buffer. After it has returned with E_DHD_OK this buffer is owned by the application and has to be released (using dhd_free_buffer) at some point later. Furthermore the function will modify the internal driver state, so use dhd_getstate/dhd_setstate to backup and restore the original driver state!

Parameters
Parameters 
Description 
a_device 
Driver context handle (as returned by dhd_init). 
a_input 
Input buffer struct (must have a valid m_format/m_bpp members) 
a_output 
Output buffer struct (must have valid m_format/m_bpp members) 
a_mode 
Desired memory layout for the output (E_DHD_ACCESS_LINEAR, E_DHD_ACCESS_SWIZZLE_4, E_DHD_ACCESS_SWIZZLE_16 or E_DHD_ACCESS_SWIZZLE_FULL) 
Returns

E_DHD_OK if successful, E_DHD_ERROR_INVALID_ARGUMENT in case of an invalid input parameter, E_DHD_ERROR_NO_MEMORY if there was not enough video memory or E_DHD_ERROR_INTERNAL if internal structures were inconsistent
