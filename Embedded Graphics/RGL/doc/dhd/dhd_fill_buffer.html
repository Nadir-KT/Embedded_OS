D/AVE HD Driver - Internal Documentation
dhd_fill_buffer
Rendering Functions | See Also | Send Feedback
Summary

Fill an entire buffer with arbitrary data.

Syntax
dhd_enum_t dhd_fill_buffer(dhd_handle_t a_device, const dhd_buffer_t * a_buffer, dhd_uint32_t a_data, dhd_uint32_t a_mask);
Description

The function is equivalent to a dhd_fill operation on the full buffer surface and a subsequent dhd_wait_fill. Therefore other draw operations can be issued directly after a fill_buffer call, without the need for additional synchronization.

Parameters
Parameters 
Description 
a_device 
Driver context handle (as returned by dhd_init) 
a_buffer 
Pointer to an allocated dhd_buffer_t (see dhd_alloc_buffer) 
a_data 
Pixel data to fill into the memory (raw bits must match the target pixel organization). 
a_mask 
One bit per byte write enable. To write all bytes use 0xf. 
Returns

E_DHD_OK if successful E_DHD_ERROR_INVALID_ARGUMENT in case of an invalid input parameter E_DHD_ERROR_NO_BUFFER the buffer is invalid or has not yet been allocated
