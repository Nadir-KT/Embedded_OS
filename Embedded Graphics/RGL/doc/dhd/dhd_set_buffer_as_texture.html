dhd_set_buffer_as_texture
State Utility Functions | See Also | Send Feedback
Summary

Set texture address, format, size and wrapmode from buffer.

Syntax
dhd_enum_t dhd_set_buffer_as_texture(dhd_handle_t a_device, dhd_buffer_t * a_buffer);
Description

Selecting a texture usually requires a series of state changes (see e.g. dhd_set_texture_address, dhd_set_texture_format, dhd_get_texture_size). With this function all relevant states can be changed according to the content of the given buffer. For this to work the optional m_format member of dhd_buffer_t has to be specified and if the given format is a texture format (dhd_texel_org_t) the optional members m_access, m_uaxis_wrap and m_vaxis_wrap also have to be specified. 

 

If the buffer is describing a framebuffer or zsa buffer (determined by it's m_format member containing a type from dhd_pixel_org_t or dhd_depth_org_t) the function will automatically choose the texture format that matches the data best. In this case m_access, m_uaxis_wrap and m_vaxis_wrap are ignored and assumed to be E_DHD_ACCESS_LINEAR and E_DHD_TEX_CLAMP. 

 

When mapping a zsa buffer the function will try to map stencil to the alpha channel, global alpha to the red channel (when present) and depth to the remaining channels.

Parameters
Parameters 
Description 
a_device 
Driver context handle (as returned by dhd_init). 
a_buffer 
Pointer to an allocated dhd_buffer_t with m_format description (see dhd_alloc_buffer) 
Returns

E_DHD_OK if successful E_DHD_ERROR_INVALID_ARGUMENT in case of an invalid input parameter E_DHD_ERROR_NO_BUFFER if the buffer is invalid or has not yet been allocated E_DHD_ERROR_NOT_SUPPORTED if the buffers m_format member does not contain a valid format id
