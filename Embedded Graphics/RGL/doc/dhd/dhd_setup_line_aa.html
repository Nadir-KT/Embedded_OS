dhd_setup_line_aa
Setup Utility Functions | Send Feedback
Summary

Setup wide or antialiased line geometry.

Syntax
dhd_bool_t dhd_setup_line_aa(dhd_handle_t a_device, dhd_fx28_4_t a_x1, dhd_fx28_4_t a_y1, dhd_fx28_4_t a_x2, dhd_fx28_4_t a_y2, dhd_uint32_t a_width, dhd_prim_data_t * a_pout, dhd_edge_data_t * a_eout, dhd_util_clipmode_t a_clipmode);
Description

Antialiased lines support an arbitary width by default. The function will use four edges (DHD_EDGE0 to DHD_EDGE3). The first two are parallel to the line and each offset by width/2 pixels. The last two are end caps perpendicular to the base line. 

Parameters a_pout and a_eout are not optional and can not be null. 

 

By default the line end caps pass directly through the specified endpoints. In order to extend them by width pixels outwards simply do this after setup:

Copy Code
a_eout[2].m_start += a_width << (18 - 4); // convert width from 4bit to 18bit fixedpoint
a_eout[3].m_start -= a_width << (18 - 4);

 

If for some reason the line should be rendered without antialiasing, simply modify the edgecontrol after setup like this:

Copy Code
a_pout->m_edgecontrol |= DHD_EDGE0(E_DHD_IEDGE) | DHD_EDGE1(E_DHD_IEDGE) | DHD_EDGE2(E_DHD_IEDGE) | DHD_EDGE3(E_DHD_IEDGE);
Parameters
Parameters 
Description 
a_device 
Driver context handle (as returned by dhd_init). 
a_x1 
X position of first point (4bit subpixel precision) 
a_y1 
Y position of first point (4bit subpixel precision) 
a_x2 
X position of second point (4bit subpixel precision) 
a_y2 
Y position of second point (4bit subpixel precision) 
a_width 
Linewidth (4bit subpixel precision) 
a_pout 
Pointer to dhd_prim_data_t receiving bounding box and edge control flags 
a_eout 
Pointer to dhd_edge_data_t[4] receiving edge information 
a_clipmode 
Specify to what region the geometry shall be clipped (see dhd_util_clipmode_t) 
Returns

DHD_TRUE if resulting line is visible DHD_FALSE if resulting line was entirely clipped and should not be drawn
