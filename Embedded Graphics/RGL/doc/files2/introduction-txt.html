Graphics library (RGL)

The graphics library (RGL) is provided as following path.

vlib/device/d1mx/lib/libd1mx_rh850_ghs.a

The library consists of generic drivers for

  • the 2D graphics drawing engine (Drw2D)
  • the Window Manager (WM) abstraction layer for Video-In, Video-Out and Sprites
  • the Video-In, Video-Out driver (VDCE)
  • the Sprite driver (SPEA)
  • the Video warping engine driver (VOWE)
  • the JPEG decoding driver (JCUA)
  • the Serial flash driver (SFMA)
  • the Hyper Bus driver (HYPB) - only D1M1A device or D1M1-V2 device
  • the Octa Bus driver (OCTA) - only D1M1A device or D1M1-V2 device
  • the Open VG (OVG) - only optional RGL package
  • the Display Output Comparator driver (DISCOM)
  • the Video Output Checker driver (VOCA)

Porting layer

Those generic drivers require a hardware abstraction also called later “support functions” or “Porting layer”.

Those functions are device or OS dependent, included as source example but not part of the library.

Porting layer is not target of quality assurance.

The user is free to modify this part of the code to fit its application.

The documentation for the porting layer is

  • RGL-D1x_Porting-Layer-Guide.pdf

Example code

RGL package includes sample applications to guide the usage of RGL.

Sample applications are not part of the RGL and not target of quality assurance.

Refer to the readme_rgl.txt about detail of applications.

The documentation for the one sample application is

  • Grape_app_UM.pdf

The RGL package includes example code which are required to run the delivered applications.

Those example codes are not part of the RGL and not target of quality assurance.

Generally, the following drivers are included.  The list is not exhaustive.

  • Board support package (BSP)
  • Sample OS (ROS)
  • CDI (common device interface. includes a simple memory manager)
  • Port controller driver
  • Interrupt controller driver
  • Clock controller driver
  • XBus driver
  • I2C/CSI/GPIO driver
  • Timers driver (e.g.  OSTM)
  • NAND Flash (NFMA) driver
  • Error Control Module (ECM) driver

Caution of required compiler option

Caution

Atomic bit manipulation commands (SET1, CLR1, TST1 and CAXI) cannot be used on Video-RAM or external SD-RAM because they are ignored by the AXI-Bus.

If you need to access above mentioned RAM, the following compiler option must be used

-no_rh850_1bit

The RGL itself is compiled with the option.  See the readme.txt for other compiler options.

Caution of interrupt

Caution

Due to an issue within the CPU (G3M core) an application shall never call almost functions of the RGL API from within the following interrupts: EIINT, SYSERR, FENMI/FEINT, FPP/FPI

Please refer to the User’s Manual “Call from Interrupt” for the availability of each function.

Within the delivered sample applications, this rule is adhered to.
